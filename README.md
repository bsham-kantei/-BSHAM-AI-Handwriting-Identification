# 運動生成モデルに基づく次世代筆者識別AI（BSHAM™）技術仕様書

## 第1章：背景と既存AIモデルの論理的破綻（Problem Statement）

### 1-1. 形態比較型AIの「汚染」と限界
現代の画像認識AI（CNN：畳み込みニューラルネットワークや、ViT：ビジョン・トランスフォーマー等）を用いた筆跡鑑定システムの多くは、重篤な設計上の「汚染」を抱えている。それは、入力データとして文字の「形状（2次元の静的なピクセルデータ）」のみに依存し、結果としての外形的類似性をスコアリングしている点である。

言語は社会的な共通記号であるため、異なる書字手が記述した場合であっても、一定のフォント的枠組み（形状の収束性）から逸脱しない。したがって、形状特徴量空間において高い類似度（ハネ、ハライ、文字の大きさ、傾きの合致等）が検出されたとしても、それは以下の3つの対立仮説（Alternative Hypotheses）を原理的に排除できない。

1. **同一の運動プログラム（本人）による自然な書字**
2. **他者による視覚依存の高度な模写（偽筆）**
3. **言語体系の共通性に基づく幾何学的な偶然の一致**

模写者は文字の「外形（絵）」を認識し、それを手先でトレースするため、視覚的な結果としての「形状」は本人のものと極めて酷似する。外形的類似性の比較に基づく既存 of AIモデルおよび伝統的鑑定手法は、これら3つの因果の異なる事象を識別する能力（対立仮説の排除能力）を数学的・生理学的に有しておらず、司法の場におけるエビデンスとしての証明力を欠いている。形状的特徴は、筆者識別において「評価対象から排除されるべき汚染情報」として定義されなければならない。

### 1-2. 司法適合性（Explainable AI: XAI）の欠如と限界論の解体
既存のディープラーニングモデルは、数百万のパラメータによる複雑な非線形変換を経て「同一人確率：95%」といったソフトマックス確率をブラックボックスから出力する。しかし、このアプローチは日本の司法最高峰が要求する検討枠組みと完全に衝突する。

日本の裁判実務、とりわけ**平成12年東京高裁判決**において示された「筆跡鑑定の限界論（異筆断定への慎重論）」の本質は、「筆跡鑑定」という行為そのものの否定ではない。検証対象となった当時の主流手法、すなわち「鑑定人の主観的経験則、肉眼による印象評価、恣意的な形態比較」という、**判断過程の不透明さと第三者による検証可能性（再現性）の欠如**に対する合理的な排斥である。

さらに、**令和3年仙台高裁判決**においては、以下の3点が明確な司法の判断枠組みとして定立された。
1. **原本性の厳格な要求**（カーボン複写等、改変可能性のある資料の証拠価値の制限）
2. **恣意的な特徴抽出の排除**（鑑定人が都合の良い部位だけを選択して比較することの禁止）
3. **指摘基準の明示**（なぜ一致・不一致としたのかを第三者が完全に追跡できる検証可能性の担保）

既存のブラックボックス型画像認識AIは、まさにこの高裁が指し示した「形態比較型の欠陥」をデジタル上で再現しているに過ぎず、法科学的証拠能力を持たない。次世代の筆者識別AIが司法適合性を得るためには、確率数値の独り歩きを厳格にロックし、判断の全プロセスを数理的・生理学的な因果関係として可視化する「説明可能なAI（XAI）」へのパラダイムシフトが必要不可欠である。

## 第2章：運動生成識別AIのコアアーキテクチャ（Core Architecture）

### 2-1. 逆ダイナミクス推定による運動制御パラメータの復元
BSHAM™（Brain Science Handwriting Audit Method）アーキテクチャの根幹は、筆跡を静的な「図形」ではなく、**「脳の運動野からの指令、小脳による補正、大脳基底核による自動化制御が、執筆者の身体・神経的制約を通過して紙面に残された『動的運動出力の痕跡』」**として再定義する点にある。

本AIシステムは、前段のエンコーダー層において、2次元の静止画入力（ピクセル空間）から、ペン先が描いた物理的な時間軸（t）を含む**3次元時系列運筆ベクトル（X, Y, t）を逆ダイナミクス計算（Inverse Dynamics Estimation）によって復元**する。

模写者が視覚依存のフィードバック制御（文字の形を目で追いながら、慎重に速度を落として書く行為）を行った場合、運筆軌道における躍度（Jerk：加速度の微分値）が異常に増大し、滑らかな加減速波形（物理的な運動指令パルス）が不連続に破綻する。AIは、文字の形状ではなく、この復元された運動生成プロセスの滑らかさと一貫性を物理演算として評価する。

### 2-2. 5つの独立・準独立神経運動制御モジュール（5-Axis Control Units）
AIの深層ウェブにおいて特徴量を一括圧縮（ブラックボックス化）することを防ぐため、システム内部に人間の脳の制御系に対応した**「5つの独立または準独立な解析エージェント」**を配置する。

1. **Ⅰ. 距離制御ユニット（空間配置・位置関係系）**
   文字内の点画の長さ、画と画の間隔、相対的なサイズ、空間的な位置距離の制御を幾何学的に測定する。
2. **Ⅱ. 方向制御ユニット（軌跡形成・運動経路系）**
   点画の傾き、アプローチ角度、および運筆時の動的な方向ベクトルの安定性をベクトル曲率として抽出する。
3. **Ⅲ. 連結制御ユニット（接続・連結構造系）**
   二つのストロークが接合、交差、接続する際の、空間認知に基づく接続構造（交差位置のズレの固有パターン等）を解析する。
4. **Ⅳ. 連鎖制御ユニット（運動連鎖・協調制御系）**
   特に「ひらがな連続（連綿）」等における、視覚フィードバックが追いつかない超高速な自動化運動（フィードフォワード制御）の連鎖構造と、協調運動の連続性を時系列解析によって検出する。模写者が最も擬態困難な脳の自動化プログラム領域である。
5. **Ⅴ. 終端制御ユニット（起筆・終筆および運動開始停止制御系）**
   画の書き始め（起筆）と書き終わり（終端）における、ペン先の微細な加減速プロファイル（停止タイミング、収束構造、力の抜き方）を物理シミュレーションによって判定する。

**独立性および準独立性の定義：**
これら5つの異なる分類に属する特徴は、脳内の異なる運動制御系に由来するため、相互に完全に**「独立した特徴」**として処理する。また、同一の分類内（例：同じ距離制御）であっても、「画の長さ」と「交差位置」のように、制御対象や監査部位が異なる場合は**「準独立特徴」**として定義し、後段の確率計算における二重カウント（過学習バイアス）を排除するマトリクス構造を形成する。

## 第3章：階層型データガバナンスと前処理（Data Validation Pipeline）

### 3-1. 对照資料における「偽造混入」の自動検知と排除（インターロック）
従来の形態比較型鑑定、および既存の画像認識AIにおける最大のバグは、「対照資料（執筆者本人のものとされる基準資料）は100%真正である」という盲信（無批判な前提）に依存している点である。

実際には、対照資料の中に「カーボン複写の上からボールペンでなぞった上書き」や「他人の精巧な模写（偽筆）」が巧妙に紛れ込んでいるケースがあり、これを無警戒に学習・比較対象に組み込むことで致命的な誤鑑定（サンプリング汚染）が発生する。

BSHAM™アーキテクチャでは、データ入力フェーズにおいて以下の**「対照資料スクリーニングゲート」**を実装し、短時間で基準資料の健全性を監査する。

1. **上書き・トレースの動的波形検知**
   カーボン複写へのボールペン上書きやトレース行為は、肉眼や単純な画像解析で形状が一致していても、運筆の速度・加速度・ジャーク（躍度）プロファイルが「視覚依存のフィードバック運動」となるため、物理的に必ず破綻する。
2. **能力・反復の一貫性チェック**
   対照資料として入力された複数の文字群を、後述する Layer 1（能力）および Layer 2（反復）のアルゴリズムにかけ、固有の運動プログラムから逸脱した「異質な書字構造（偽造の混入）」を統計的な外れ値として検出する。
3. **結論**
   対照資料にわずかでも不適合な運動構造が認められた場合、AIはそれを「真正な基準データ」として認めず、判定パイプラインの手前で自動的に排除・隔離する（Early Stopping）。これにより、サンプリング汚染による誤判定を根本から防止する。

### 3-2. ドメインノイズ（自然変動）とシグナル（安定反復構造）の分離
書字運動には、執筆時期、身体・精神状態、疾病、筆記具などの外部環境変化による「状態差（一時的な自然変動）」が必ず発生する。AIはこれらをエラー（別人の証拠）と誤認してはならない。

システムは、表面的なバリエーション（ドメインノイズ）をデノイジング（消去）し、脳に長期的に保持される運動学習の核心（シグナル）のみを抽出するため、不変の運動計画パラメータ（速度と曲率のベキ乗則等）を特徴量として固定するロバスト（頑健）設計を採用する。

## 第4章：厳格な階層型意思決定および数理評価システム（Mathematical & Decision Engine）

### 4-1. 順序反転不可能な3ステップ・パイプライン
本AIシステムは、抽出された特徴の類似度を単に足し算・平均化するブラックボックス計算を完全に排除する。意思決定レイヤーは、以下の逆転不可能な3つの階層（Layer）を直列に結合した　プログラム制御を行う。この実行順序は判定条件であり、いかなる場合も逆転してはならない。

#### 【Layer 1：能力評価ゲート（成立条件）】
* **アルゴリズム**：対象人物の身体的・神経的条件（運動機能の限界、低下状態等）をコード化し、検証対象の筆跡が物理的・生理学的に「生成可能（書けるか）」を検証する。
* **評価対象**：運動の滑らかさ、筆圧の安定性、連続運動の成立、書字速度の自然性。
* **処理**：能力的に不可能、または著しく不自然な運動軌跡（低ジャークな超高速連続運動等）が検出された場合、AIは形状の一致度にかかわらず、以降の計算をすべてシャットダウン（Early Stopping）し、即座に**「能力不適合による不成立（排除）」**を出力する。

#### 【Layer 2：構造・反復評価層（証拠構造）】
* **アルゴリズム**：Layer 1を通過した運動制御特徴（距離、方向、連結、連鎖、終端）の出現率を、以下の数式に基づき自動カウントする。
  $$出現率 = \frac{m}{n}$$
  * $n$：当該特徴が確認可能であった全出現箇所の総数（機会数。鑑定人の恣意的なチェリーピッキングを排し、同一文字の全出現箇所および同一構造を有する類似範囲から網羅的に全自動サンプリングする）。
  * $m$：当該特徴が実際に確認された回数（出現数）。
* **ハードしきい値（75%ルール）**：
  人間の運動出力に伴う自然変動を許容しつつ、安定したプログラムの再現性を担保するため、**出現率が75%（4回中3回）以上の特徴のみを「安定特徴」として採択**する。75%未満の単発的・低頻度な特徴は、偶発的揺らぎ（ノイズ）として判定材料から強制的に「棄却（重みをゼロにする）」する。

#### 【Layer 3：数理的補強評価層（確率統計）】
Layer 2で採択された安定特徴、および非再現構造の同時成立可能性を定量的に補強するため、以下の厳格な数理公式を適用する。

##### ① ラプラス（Laplace）推定による小標本確率補正
データ数が限られている条件下（小標本）での不確実性を数学的に補正し、AIによる確率の過大評価・過学習（ゼロ頻度問題）を防ぐため、次回出現可能性を以下のラプラスの継承公式で保守的（安全側）に推定する。
$$P = \frac{m + 1}{n + 2}$$

##### ② 複数独立特徴における「同時非再現構造」の積算モデル（積の法則）
単一の特徴の不一致は偶然でも発生し得るが、脳の独立した異なる運動制御系（最大5分類）や、制御対象の異なる準独立特徴において**同時に不一致（非再現構造）が成立する確率（$P_{total}$）**は、各特徴の非再現確率の「積」として指数関数的に低下する。　
$$P_{total} = \left( 1 - \frac{m + 1}{n + 2} \right)^c$$
* $c$：異なる運動制御過程または神経運動制御系に基づく独立・準独立分類のうち、両資料間で反復構造が非再現（不一致）となった分類数（最大5）。

$c$（破綻している制御次元）が増加するほど、$P_{total}$ の値は極限までゼロに接近する。AIはこの数理的検定により、「同一の運動プログラム（本人）からこの破綻が偶然同時に生じることは数学的に説明不可能である」ことを客観的に証明し、対立仮説を完全に排除する。

##### ③ 非対称構造（排他的反復性・排他的交差反復性）の検出
模写者が文字の形を意識的に真似ようとした結果、資料内に模写者自身の無自覚な自動化運動プログラムが浮き出てしまう現象を、AIが「排他的交差反復マトリクス」として検出する。
* **排他的反復性**：一方の資料でのみ安定して反復する特徴が、他方では全く反復しない構造。
* **排他的交差反復性**：両資料で、それぞれ異なる特徴が反復して対立している構造。
これらの非対称構造が認められる場合、単一の運動プログラムによる生成としては整合しないため、同一人仮説を100%論理排斥（Reject）する。

## 第5章：出力仕様と検証可能性（Audit Log Output）

### 5-1. 二者択一の断定的ディシジョン（グレーゾーンの排除）
AIの最終出力層は、従来のAIのように「同一人である確率：72%」といった曖昧なソフトマックス確率の数値をそのまま表示して判断を人間に丸投げすることを厳格に禁止する。

Layer 1からLayer 3までの論理的・数理的検証条件がすべて完全クリアされた場合のみ、システムはロジックベースで以下のいずれかを**二者択一で断定的（逃げのない結論形式）**に出力する。
* **判定：TRUE（同一人の筆跡である）**
* **判定：FALSE（別人の筆跡である）**

### 5-2. 司法提出用「筆跡監査ログ（Audit Trail）」の自動生成
AIのブラックボックス化を100%排除し、裁判官、弁護士、および第三者が判断の全過程を完全に追跡・再現（監査）できるよう、システムは結論と同時に以下の項目を網羅した「Markdown形式の筆跡監査報告書（Audit Log）」を自動生成・出力する。
1. **対照資料の適格性・健全性監査ログ**（偽造混入排除 of 判定根拠）
2. **網羅的サンプリングログ**（全文字からの機会数 $n$ と出現数 $m$ のカウント全履歴）
3. **独立特徴分類マトリクス**（どの運動制御系が一致・不一致となったかの可視化データ）
4. **数理計算プロセスの明示**（ラプラス推定の適用、および積の法則による $P_{total}$ 計算式の全展開）
